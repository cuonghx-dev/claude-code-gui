# ADR 0006 — ts-rs export: committed types + drift gate

**Status**: Accepted
**Date**: 2026-05-10

## Context

Rust IPC types must be available to the TypeScript frontend. Options: (1) generate at build time from a script the frontend runs, (2) commit generated types and verify in CI, (3) hand-maintain TS types and accept drift.

## Decision

Option 2 — commit generated `.ts` files under `frontend/src/types/ipc/` and gate drift in CI.

Mechanism:

1. Every IPC type derives `ts_rs::TS` with `#[ts(export, export_to = "../../../../frontend/src/types/ipc/")]` (path relative to crate `Cargo.toml`).
2. `crates/core/tests/export_types.rs` and `crates/app/tests/export_types.rs` exist with empty bodies — `cargo test` materializes the `.ts` files via the derive's export hook.
3. CI runs:
   ```
   cargo test --workspace --tests export_types_no_drift
   git diff --exit-code -- frontend/src/types/ipc/
   ```
4. The barrel file `frontend/src/types/ipc/index.ts` is regenerated by `frontend/scripts/build-ipc-barrel.ts` invoked by the drift test before the `git diff`.
5. Generated files are marked `linguist-generated=true` in `.gitattributes` so GitHub UI doesn't surface them in diffs.

## Phase 0 done-criterion

The drift gate must catch at least one real drift in CI before Phase 0 closes. Intentionally introduce a Rust type change without re-running `cargo test`, verify CI fails, then revert.

## Consequences

- Frontend builds don't require `cargo`.
- One `cargo test` run is the regeneration command — no separate codegen script.
- Adds `cargo test` to the contributor inner loop on Rust type changes.
